- **單一專案**：儲存庫根目錄的 `src/`、`tests/`

## 相依性與執行順序

### 階段相依性

- **設置（階段一）**：無相依性 - 可立即開始
- **基礎建設（階段二）**：依賴設置完成 - 阻塞所有用戶故事
- **用戶故事（階段三+）**：全部依賴基礎建設階段完成
  - 用戶故事可以並行進行（如果有足夠人力）
  - 或依優先級順序執行（P1 → P2 → P3）
- **優化（最終階段）**：依賴所有預期用戶故事完成

### 用戶故事相依性

- **用戶故事一（P1）**：基礎建設（階段二）完成後可開始 - 不依賴其他故事
- **用戶故事二（P2）**：基礎建設（階段二）完成後可開始 - 與 US1 整合但可獨立測試
- **用戶故事三（P3）**：基礎建設（階段二）完成後可開始 - 與 US1/US2 整合但可獨立測試

### 每個用戶故事內部

- 組合式函數在組件之前（資料管理在 UI 之前）
- 組件在整合之前（個別組件在 App.vue 整合之前）
- 核心功能在持久化之前（基本功能在 localStorage 之前）
- 故事實作在與其他故事整合之前

### 並行機會

- 所有標記為 [並行] 的設置任務可以並行執行
- 所有標記為 [並行] 的基礎建設任務可以並行執行（在階段二內）
- 基礎建設階段完成後，所有用戶故事可以並行開始（如果團隊容量允許）
- 在每個用戶故事內，標記為 [並行] 的任務可以並行執行
- 不同的用戶故事可以由不同團隊成員並行開發

## 實作策略

### MVP 優先（僅用戶故事一）

1. 完成階段一：設置
2. 完成階段二：基礎建設（關鍵 - 阻塞所有故事）
3. 完成階段三：用戶故事一
4. **停止並驗證**：獨立測試用戶故事一
5. 部署/展示基本題目瀏覽功能

### 漸進式交付

1. 完成設置 + 基礎建設 → 基礎就緒
2. 添加用戶故事一 → 獨立測試 → 部署/展示（MVP：基本題目瀏覽）
3. 添加用戶故事二 → 獨立測試 → 部署/展示（添加分頁導航）
4. 添加用戶故事三 → 獨立測試 → 部署/展示（完整雙語支援）
5. 每個故事都在不破壞現有故事的情況下增加價值

### 並行團隊策略

多位開發人員：

1. 團隊一起完成設置 + 基礎建設
2. 基礎建設完成後：
   - 開發人員 A：用戶故事一（核心題目顯示）
   - 開發人員 B：用戶故事二（分頁系統）
   - 開發人員 C：用戶故事三（語言切換）
3. 故事獨立完成並整合

## 備註

- [並行] 任務 = 不同檔案，無相依性，可安全並行執行
- [故事] 標籤將任務對映到特定用戶故事以便追蹤
- 每個用戶故事都應該可以獨立完成和測試
- 所有組合式函數專注於 Vue 3 Composition API 模式
- 所有實作使用 TypeScript 嚴格模式
- 所有樣式遵循 SCSS 模組化架構
- 使用 localStorage 保持用戶狀態（頁面、語言、摺疊狀態）
- 確保響應式設計在行動裝置、平板和桌面上都能運作
- 每個任務或邏輯任務組後提交
- 在任何檢查點停止以獨立驗證故事
- MVP 可以只實作用戶故事一，已足夠提供核心學習價值
